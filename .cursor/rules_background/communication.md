# Communication Rules 背景

このドキュメントは[communication.mdc](./../rules/communication.mdc)のルールの背景を説明するものです。

## 1. すべての回答は日本語で行う

- **背景**: プロジェクトメンバーが日本語を母国語とするため、コミュニケーションコストを最小化し、正確な理解を促進する。
- **効果**: 誤解を減らし、レビューや議論を円滑にする。

## 2. 進捗報告や設計共有では要点を簡潔にまとめ、必要ならTDDの観点を添える

- **背景**: 長文の報告は読まれず、重要な情報が埋もれる可能性がある。また、このプロジェクトではTDDを採用しているため、テストの観点を明示することで品質を担保する。
- **効果**: 報告の可読性が向上し、レビュアーが素早く理解できる。TDDの観点を含めることで、テスタビリティを意識した設計が促進される。

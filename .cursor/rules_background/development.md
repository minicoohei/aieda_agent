# Development Process Rules 背景

このドキュメントは[development.mdc](./../rules/development.mdc)のルールの背景を説明するものです。

## 1. TDD（テスト駆動開発）の採用

- **背景**: 品質の高いコードベースを維持し、リファクタリングを安全に行うため。
- **効果**: バグの早期発見、テスタビリティの高い設計、リグレッションの防止。

## 2. Serena MCPの活用

- **背景**: コードベースの構造把握や影響範囲の調査を手動で行うと時間がかかり、見落としが発生する可能性がある。
- **効果**: シンボリック系ツールを使うことで、正確かつ迅速にコードベースを理解できる。

## 3. Context7での公式ドキュメント参照

- **背景**: 最新のAPI仕様やベストプラクティスを確認せずに実装すると、非推奨の方法を使ったり、新しい機能を見逃したりする可能性がある。
- **効果**: 公式ドキュメントを参照することで、正確で最新の情報に基づいた実装が可能になる。

## 4. 既存の変更を元に戻さない

- **背景**: 過去の変更には理由があり、勝手に元に戻すと意図しない問題を引き起こす可能性がある。
- **効果**: 安全性を確保し、チーム内での信頼を維持する。

## 5. 環境変数の一時的な上書き

- **背景**: `GOOGLE_APPLICATION_CREDENTIALS`などの環境変数を常設で書き換えると、他のプロジェクトに影響を与える可能性がある。
- **効果**: プロジェクト間の干渉を防ぎ、安全な開発環境を維持する。
- **実例**: `env -u GOOGLE_APPLICATION_CREDENTIALS python script.py` または `GOOGLE_APPLICATION_CREDENTIALS=/path/to/key.json python script.py`

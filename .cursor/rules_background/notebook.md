# Notebook Usage Rules 背景

このドキュメントは[notebook.mdc](./../rules/notebook.mdc)のルールの背景を説明するものです。

## 1. Marimoを使用

- **背景**: Jupyter Notebookは実行順序が不定で再現性に問題がある。Marimoはリアクティブな実行モデルで、セルの依存関係を自動管理し、常に一貫した状態を保つ。インタラクティブUIコンポーネント（スライダー、ドロップダウン等）も充実している。
- **効果**: 再現性の高い分析が可能になり、インタラクティブな探索が容易になる。
- **実例**: `marimo edit notebooks/your_analysis.py --port 4173`

## 2. 分析目的を明記

- **背景**: 後からNotebookを見返したときに、何のための分析だったかわからなくなる。
- **効果**: 目的が明確になり、Notebookの再利用や共有が容易になる。

## 3. 仮想環境の使用とedit_notebookツール

- **背景**: グローバル環境にパッケージをインストールすると、バージョン競合や環境汚染が発生する。また、Notebookの編集は専用ツールを使うことで、セルの追加・削除・編集を正確に行える。
- **効果**: 環境の分離と再現性を確保し、Notebookの編集ミスを防ぐ。

## 4. セルには英語ラベル、説明は日本語

- **背景**: コード内の変数名やセルIDは英語が標準的だが、説明は日本語の方が理解しやすい。
- **効果**: 国際的な標準に従いつつ、チーム内でのコミュニケーションを円滑にする。

## 5-8. ファイル整理、エラー対応、データアクセス、テンプレート

- **背景**: 統一された方法論で作業することで、効率性、安全性、一貫性を確保する。
- **効果**: 効率的な作業、知識の蓄積、セキュリティの確保、一貫性のある出力を実現する。

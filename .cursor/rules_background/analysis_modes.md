# Analysis Modes 背景

このドキュメントは[analysis_modes.mdc](./../rules/analysis_modes.mdc)のルールの背景を説明するものです。

## データ分析の2つのモード

- **背景**: データ分析には「結果を他者に伝える」目的と「自分で探索して理解を深める」目的があり、それぞれに適したツールと手法が異なる。両者を混同すると、非効率な作業になったり、適切な出力が得られなかったりする。
- **効果**: 目的に応じた最適なツールと手法を選ぶことで、効率的かつ効果的な分析が可能になる。

## エビデンス提示モード

- **背景**: 他者に分析結果を伝える際は、一目で理解できる画像（グラフ）が最も効果的。数値だけの表は読みにくく、理解に時間がかかる。
- **効果**: 画像を多用することで、説得力のあるレポートを作成でき、意思決定を支援できる。
- **実例**: 月次レポート、経営陣へのプレゼン、マルチエージェントでの報告書作成。

## 探索モード

- **背景**: 静的なレポートを見て「この部分をもっと詳しく知りたい」と思っても、再度スクリプトを編集して実行するのは手間がかかる。インタラクティブなツールを使うことで、リアルタイムでパラメータを変更し、自由に探索できる。
- **効果**: Marimoを使うことで、スライダーやドロップダウンで動的にフィルタリング・集計を試行でき、新しい発見を素早く得られる。
- **実例**: 「木曜日だけ投稿が多い理由を調査」、「特定ユーザーの行動パターンを深堀り」、「新しい仮説の検証」。

## 推奨ワークフロー

- **背景**: まずエビデンス提示モードで包括的な分析を実行し、全体像を把握する。その後、興味深い発見があれば探索モードで深堀りする、という流れが最も効率的。
- **効果**: 全体像の把握→詳細探索→新しい発見の追加、というサイクルで、段階的に分析を深められる。
- **実例**:
  1. `python notebooks/00_comprehensive_eda_analysis_MAIN.py` で画像10種類+レポート生成
  2. レポートを確認し、「面白い！」と思った部分を特定
  3. `marimo edit notebooks/eda_sakurai_yui.py --port 4173` で深堀り分析
  4. 新しい発見があれば、再度画像を生成してレポートに追加
